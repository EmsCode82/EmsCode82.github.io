Assetgineer Live

Assetgineer is an AI-assisted tool for turning high-level game concepts into consistent visual assets, then managing those assets with tags, collections, and version history.

Contents: Overview · Prompt Engine · Asset Management · Screens

Overview

Moving beyond traditional pipelines, Assetgineer turns ideas into production-ready assets across characters, locations, items, weapons, enemies, VFX, and UI. It emphasizes consistency of style and lore as a project grows.

Intelligent Prompt Engine

A structured prompt generator translates user inputs into model-ready prompts for reliable output.

  • Contextual controls: races/biomes/architecture for world coherence; elemental infusions for items/weapons.
  • Adaptive styling: pixel art → photoreal; positive/negative terms auto-tuned to the chosen aesthetic.
  • Granular knobs: physical attributes, outfit slots/materials/colors, staging (camera/lighting/bg), and pose.
  • Randomize intelligently: “randomize all” or section-only to iterate quickly without losing context.
  • Image-to-image & in-painting: refine existing images or surgically edit masked regions.
  • Prompt Analysis: a tree view shows how the final prompt is composed, step-by-step.

Comprehensive Asset Management

  • Dynamic library: browse, filter, and sort by type, tags, and custom collections.
  • Collections: group related assets for features or releases.
  • Version history: variations, diffs, and quick rollback.
  • Bulk ops: tag, assign, and delete at scale.
  • Details view: image, original prompt, parameters, lineage.
  • Loop-back: load any asset back into the generator for further edits.

Learning notes

Prompt engine anatomy (inputs → prompt)

Structured inputs (category, style, constraints, negatives) are combined into a stable prompt so generated images stay consistent across a project. The Prompt Analysis view shows the composition tree so you can reason about changes.

Versioned asset library

Treat art like code: store parameters, prompt, and outputs together; create variations; compare; and roll back when a direction doesn’t land. Tags/collections keep sets coherent for features.
